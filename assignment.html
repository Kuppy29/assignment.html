(6) What is the class attribute in HTML?

One or more classnames are specified for an element by the class attribute. In a style sheet, the class attribute is typically used to identify a class. However, it can also be used by JavaScript (via the HTML DOM) to modify HTML components that belong to a particular class.

Example:

< div class="name" >

As per the above example the 'class' is defined to 'name' and on the webpage the child elements of that class could be divided by using the additional properties of CSS.

Question 19:



(19) What is difference between HTML and XHTML?

The standard markup language used to create web pages is called HTML, and the more standardised form of HTML is called XHTML. Numerous features, including support for multimedia, style, and scripting, are present in both HTML and XHTML.
HTML stands for Hypertext Markup Language. XHTML stands for Extensible Hypertext Markup Language.

HTML VS. XHTML

HTML XHTML
1. HTML is an SGML-based language. That is, it defines the standard for generalizing the markup languages for documents. SGML stands for Standard Generalized Markup Language. 1. XHTML is an XML-based language, which means it manipulates and processes data using XML technologies.
2. HTML is not a case-sensitive language. 2. XHTML is a case-sensitive language.
3. The tags and attributes in HTML can be specified either in lowercase or uppercase since it is not case-sensitive. 3. All XHTML elements and attributes must be in lowercase since it is a case-sensitive language.
4. The HTML document requires a minimum of four tags to create an HTML page that is <html>, <head>, <title>, and <body>. The <!DOCTYPE> declaration is not necessary for HTML. 4. An XHTML document must contain the <!DOCTYPE> declaration followed by the <html>, <head>, <title>, and <body> tags in its document to create a webpage. Moreover, the xmlns attribute in <html> tag is also necessary.
5. It is not mandatory to put quotes while using the attributes in HTML. 5. It is mandatory to put quotes around an attribute in XHTML.
6. The webpage is displayed even if the HTML document has some errors in it. 6. There is a more strict error handling in XHTML.
7. Some HTML elements may function properly without a closing tag. 7. All the XHTML elements must be closed. Even the empty elements also require a closing tag.
8. It can have a filename extension of.html or.htm. 8. Its filename extension can be .xhtml,.xht, or .xml.
